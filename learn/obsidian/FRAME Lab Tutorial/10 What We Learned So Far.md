# What We Learned So Far

A few lessons are holding up clearly.

## 1. The five-part shape is still strong

Facts, Rules, Map, Expect, and Acts still make sense together.
The issue was not “too many parts.”
The issue was unclear roles and awkward naming.

## 2. Simpler names help a lot

Developers read faster when the names sound normal.
This was one of the best 0.2.0 changes.

## 3. Cross-links matter more than extra fields

One owned truth plus good refs is better than repeating the same meaning in four places.

## 4. Acts is useful when it stays narrow

Acts becomes much more valuable when it acts like searchable run history.
It becomes much less valuable when it tries to summarize the whole project.

## 5. Tool-facing usefulness is part of the test

FRAME should not only read nicely as raw files.
It should also help future typed tools make better decisions.

## 6. Bloat is still the enemy

The easiest way to lose is to keep adding structure every time a fixture feels uncomfortable.
