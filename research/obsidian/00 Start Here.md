# FRAME Research Vault

This vault is the learning and research space for turning FRAME into a standard context architecture, with Haxaml as the tool built on top.

Tiny idea:

> A prompt is a sticky note. A huge context dump is a messy desk. FRAME is trying to become the labeled project cabinet. Haxaml is the tool that helps agents pull the right drawer at the right time.

## What This Vault Does Not Claim Yet

- It does not prove FRAME is already a standard.
- It does not prove the exact five-file FRAME shape is final.
- It does not say bigger context windows are useless.
- It does not say attention and context are the same thing.
- It does not say every prompt trick belongs in FRAME.
- It does not claim FRAME is already a standard.

The correct stance:

> [!summary]
> FRAME is a serious context-architecture candidate. Research 1 explains why context structure matters. Research 2 shows how prompt and context techniques can map to FRAME without turning FRAME into one giant prompt. Research 3 studies whether FRAME can become a standard context architecture, with a clear boundary between FRAME state and Haxaml tool behavior.
