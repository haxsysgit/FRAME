# Project Brain

## Tiny Idea

A project brain is the repo-owned place where agents learn the project without guessing from scattered chat history.

## The Five Questions

| Question | FRAME file |
| --- | --- |
| What is true? | `facts.yaml` |
| What must be obeyed? | `rules.yaml` |
| What happened? | `acts.yaml` |
| Where should I look? | `map.yaml` |
| What should happen next? | `expect.yaml` |

## Simple Analogy

Imagine a workshop.

- Facts are the label on the building.
- Rules are the safety signs.
- Acts are the work log.
- Map is the floor plan.
- Expect is the job board.

If a new worker walks in, they should not need to ask five people and dig through old texts.

They should read the workshop brain.

## Why It Matters

Without a project brain:

- agents repeat mistakes
- old decisions disappear
- rules drift between providers
- tests get skipped
- file ownership gets guessed
- planning lives only in chat
