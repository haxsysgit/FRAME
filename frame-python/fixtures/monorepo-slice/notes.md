# monorepo-slice fixture

Purpose:
- test that FRAME can represent partial project reality without pretending the whole monorepo is covered
- pressure-test whether map and acts are useful early parts in partial-context situations

Questions this fixture should pressure-test:
- how should partial coverage be represented?
- should partial monorepo slices be valid in strict mode or only provisional mode?
- what does a useful boundary map look like before the contract is fully mature?
