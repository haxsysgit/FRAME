# frame-python

`frame-python` is the first fresh implementation track for FRAME after the reset.

It does not exist to blindly recreate the old SDK.
It exists to test and shape the new contract.

## Current status

Scaffold only.
No active implementation has been reintroduced yet.

## What this package should eventually help answer

- what the normalized Python model should look like
- what parts of FRAME need strict validation
- what parts should stay provisional for now
- how the candidate structure behaves across real project fixtures
- where the architecture is still underspecified

## Near-term expectation

When implementation restarts, this folder should become the home for:
- package code
- package tests
- examples
- fixture-driven experiments

But the rebuild should begin from clarified architecture, not from inherited old code.
